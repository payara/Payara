To Edit a Context Service

  1. In the navigation tree, expand the Resources node.

  2. Under the Resources node, expand the Concurrent Resources node.

  3. Under the Concurrent Resources node, select Context Services.

    The Context Services page opens.

  4. On the Context Services page, click the name of the resource that you are editing.

    The Edit Context Service page opens.

    Note:

    The JNDI name is a read-only field. You can only specify the JNDI name when you create a new resource. The logical JNDI name is also a read-only field and is displayed for the default resource. See Logical JNDI Names for more information.

  5. Select the Context Information Enabled checkbox to propagate contexts to threads, and select the contexts that you want to propagate.

    This option is enabled and all contexts are selected by default. Use Shift-click or Control-click to choose multiple contexts. If this option is disabled, contexts are not propagated, even if they are selected.

  6. In the Deployment Order field, type a number to specify the deployment order of the resource.

    Resources with lower numbers are loaded first at server startup. The default value is 100.

  7. In the Description field, type a short description of the resource.

    This description is a string value and can include a maximum of 250 characters.

  8. In the Additional Properties section, do not add any additional properties.

    GlassFish Server does not define any additional properties for this resource type. Moreover, this resource type does not currently use any additional properties.

  9. Click the Target tab to manage resource targets.

    The Resource Targets page opens. The Target tab is displayed only if clusters and standalone instances have been created in the domain, in addition to the default server instance, server.

  10. On the Resource Targets page, do the following, as desired:

    1. Enable or disable defined targets for the resource.

    2. Click the Manage Targets button to add or remove targets for the resource.

  11. Click the General tab to return to the Edit Context Service page.

  12. Click Save.

To Configure the JVM General Settings

The Java Virtual Machine (JVM) enables Java byte codes to run on a physical machine. Tuning the JVM settings improves performance and avoids memory allocation errors.

The Java Virtual Machine is included in the Java Platform, Standard Edition (Java SE platform), which is required by the Payara Server. Incorrect JVM settings will prevent the server from running, so use care when changing these settings.

Note:

The terms "Java Virtual Machine" and "JVM" mean a Virtual Machine for the Java platform.

  1. In the navigation tree, expand the Configuration node.

  2. Select the configuration that you are changing.

  3. Select the JVM Settings node.

    The JVM General Settings page opens.

  4. In the navigation tree, select the Configuration node.

  5. Under the Configuration node, select the JVM Settings node.

    The JVM General Settings page opens.

  6. On the JVM General Settings page, in the Java Home field, type the name of the installation directory of the Java SE software.

    The Payara Server relies on the Java SE platform. If you enter a nonexistent directory name or the directory for an unsupported version of Java SE software, then Payara Server will not start.

  7. In the Javac Options field, type the command-line options for the Java programming language compiler.

    The Payara Server runs the compiler when Enterprise JavaBeans (EJB) components are deployed.

  8. Select the Debug Enabled checkbox to enable debugging with the Java Platform Debugger Architecture (JPDA).

    JPDA is used by application developers.

  9. In the Debug Options field, type the JPDA options to be passed to the JVM.

    This field is applicable only if debugging was enabled in the preceding step.

  10. In the RMI Compile Options field, type the command-line options for the rmic compiler.

    The Payara Server runs the rmic compiler when EJB components are deployed.

  11. In the Bytecode Preprocessor field, type a list of class names.

    Use commas to separate class names. Each class must implement the com.sun.appserv.BytecodePreprocessor interface. The classes are called in the order specified.

  12. In the Additional Properties section, specify additional properties.

    To add a property, click the Add Property button. In the blank row that appears, type the property name in the Name field, and type the property value in the Value field.

    The Payara Server does not define any additional properties for the JVM.

  13. Click Save.

    Tip:

    To return to the default settings, click the Load Defaults button, then click Save.

  14. Restart the server.

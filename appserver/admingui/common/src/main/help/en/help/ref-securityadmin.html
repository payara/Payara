Secure Administration

Use the Secure Administration page to enable or disable secure administration.

When secure administration is enabled:

  • The DAS and the instances in the domain use SSL certificates for encrypting the messages they send to each other.

  • The DAS accepts administration messages from remote admin clients such as the asadmin utility and IDEs.

The Secure Administration page contains the following information.

Enable Secure Admin

Button to enable secure administration. This button appears only when secure administration is not enabled.

Note:

If any administrative user in the domain has an empty password, attempts to enable secure administration fail.

Disable Secure Admin

Button to disable secure administration. This button appears only when secure administration is enabled.

Administration Alias

The alias that refers to the SSL/TLS certificate on the DAS. This alias is used by the DAS to identify itself to instances. The default value is s1as. This option appears only when secure administration is not enabled.

Instance Alias

The alias that refers to the SSL/TLS certificate on the instances. This alias is used by the instances to identify themselves to the DAS. The default value is glassfish-instance. This option appears only when secure administration is not enabled.

To Edit a Work Security Map

  1. In the navigation tree, expand the Resources node.

  2. Expand the Connectors node.

  3. Under the Connectors node, select the Work Security Maps node.

    The Work Security Maps page opens.

  4. On the Work Security Maps page, click the name of the work security map that you are editing.

    The Edit Work Security Map page opens.

  5. Use either the Group field or the Principal field as follows.

    • If you select the Group radio button, type a map of each back-end EIS user group to the corresponding user group defined in the GlassFish Server security domain.

      Use a comma-separated list to specify more than one mapping. For example:

      eis-group-1=server-group-1,eis-group-2=server-group-2

    • If you select the Principal radio button, type a map of each back-end EIS principal to the corresponding principal defined in the GlassFish Server security domain.

      Use a comma-separated list to specify more than one mapping. For example:

      eis-principal-1=server-principal-1,eis-principal-2=server-principal-2

  6. From the Resource Adapter drop-down list, select the resource adapter for the EIS.

  7. In the Deployment Order field, type a number to specify the deployment order of the work security map.

    Resources with a lower number are loaded first at server startup. The default value is 100.

  8. In the Description field, type a description of the work security map.

  9. Select the Status Enabled checkbox to enable the work security map.

    This option is enabled by default.

  10. Click OK.
